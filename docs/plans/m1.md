## Week 3 day 3 onward

### 3.3 — §10.5.3 GET /audit/records (NEW spec requirement)

- [ ] Add paginated-records endpoint: `GET /audit/records?after=<id>&limit=<n>`
- [ ] Response schema: `schemas/audit-records-response.schema.json` (pinned spec file, `e8b4e9e`)
- [ ] `audit:read` bearer scope; 60 rpm rate-limit; TLS 1.3
- [ ] Default limit 100, max 1000; response in chain order (earliest first)
- [ ] Pagination: `after` → `next_after` → `has_more` transitions correctly
- [ ] Not-a-side-effect: no meta-record appended on reads
- [ ] Unit tests: empty-log GENESIS marker; single-record page; multi-page; after-with-unknown-id → 404; rate-limit → 429

### 3.4 — Conformance-mode card loading (NEW spec requirement)

- [ ] Add `RUNNER_CARD_FIXTURE` env var that loads `test-vectors/conformance-card/agent-card.json` (pinned spec file, `ffe30ed`)
- [ ] At load, substitute the literal placeholder `__IMPL_REPLACES_SPKI_AT_LOAD__________________________________` in `security.trustAnchors[0].spki_sha256` with SHA-256 of the runtime signing key's SPKI. This is the ONLY field that may be rewritten.
- [ ] JCS-canonicalize the (substituted) card, sign with runtime key, serve at `/.well-known/agent-card.json` + `.jws`
- [ ] Validator now pins against DangerFullAccess card → can create three sessions at ReadOnly / WorkspaceWrite / DangerFullAccess via §12.6 tightening

### 3.5 — Conformance Tool Registry fixture loading (NEW spec requirement)

- [ ] Add `RUNNER_TOOLS_FIXTURE` env var that loads `test-vectors/tool-registry/tools.json` (pinned spec file, `e7580b9`)
- [ ] When loaded, this replaces the impl's demo fixture for the lifetime of the process
- [ ] Fixture is the conformance Tool Registry (8 tools spanning the full `risk_class × default_control` matrix). Expected 24-cell decision matrix documented in `test-vectors/tool-registry/README.md` — SV-PERM-01 validator asserts each cell.

### 3.6 — Conformance-mode bootstrap bearer (REQUIRED for POST /sessions)

- [ ] `POST /sessions` needs a bootstrap bearer for auth. For local conformance runs, support `SOA_RUNNER_BOOTSTRAP_BEARER` env var per §12.6 informative note.
- [ ] Guard: when `SOA_RUNNER_BOOTSTRAP_BEARER` is set, the server MUST bind only to loopback (reject non-loopback binds with a loud startup error). Belt-and-suspenders: fixed-bearer on public listener is a trivial admin backdoor.

### 3.7 — Tampered card boot-time rejection (REQUIRED for HR-12)

- [ ] Ensure impl's boot-time Agent Card JWS verification fails closed when fed `test-vectors/tampered-card/agent-card.json.tampered.jws` (pinned spec file, `ffe30ed`)
- [ ] Failure mode: `HostHardeningInsufficient` with `CardSignatureFailed reason=signature-invalid`; process exits non-zero OR `/ready` flips 503 with reason `card-signature-invalid`
- [ ] Validator invokes impl as subprocess with this fixture and asserts the fail-closed behavior

### 3.8 — Mismatched publisher_kid boot-time rejection (REQUIRED for SV-BOOT-01 negative path)

- [ ] Ensure impl's boot-time trust bootstrap fails closed when `test-vectors/initial-trust/mismatched-publisher-kid.json` is the active initial-trust fixture and the served card's publisher_kid doesn't match
- [ ] Failure mode: `HostHardeningInsufficient reason=bootstrap-missing`; process exits non-zero
