## M2 scope — remaining tasks

### M2-T1 — Session-file persistence with atomic writes (§12.3)

- [ ] Implement bracket-persist: for every significant event per §12.2:
  - Write `phase: "pending"` + idempotency_key + args_digest to session file BEFORE tool invocation
  - fsync the write
  - Execute tool
  - On success: update to `phase: "committed"` + result_digest, fsync again
  - On failure: update to `phase: "compensated"` with compensating action recorded
- [ ] POSIX atomic-write: `write(tmp) → fsync(tmp) → rename(tmp, final) → fsync(dir)`
- [ ] Windows atomic-write: `write(tmp) → FlushFileBuffers(tmp) → MoveFileExW(tmp, final, REPLACE_EXISTING|WRITE_THROUGH)`
- [ ] Multi-file commits use stage-activate pattern (§12.3 final bullet)
- [ ] Session file conforms to `schemas/session.schema.json` (already has `activeMode` required per L-20)
- [ ] Tests (8): POSIX fsync boundary; Windows FlushFileBuffers; write→fsync→rename→dir-fsync sequence; stage-activate commit boundary; concurrent writes serialize; partial-write detection on reopen; corrupted session-file rejection → SessionFormatIncompatible; writable-probe in /ready (already shipped, regression)

### M2-T2 — Resume algorithm (§12.5)

- [ ] `resume_session(session_id)` per §12.5:
  1. Read session file + validate `format_version == "1.0"` (else SessionFormatIncompatible)
  2. Verify `card_version` matches currently-served Agent Card; mismatch → `StopReason::CardVersionDrift`
  3. Verify `tool_pool_hash` still resolves; see §11.3
  4. For each side_effect: replay `pending` with its idempotency_key; skip `committed`; compensate/tag `inflight`; ResumeCompensationGap when not compensable
- [ ] Pre-1.0 sessions (no `activeMode` field) migrate at resume by defaulting to Agent Card's `permissions.activeMode` (L-20 migration, already shipped — regression verify)
- [ ] Tests (6): resume-pending-replays; resume-committed-skips; card_version mismatch → terminate; tool_pool_hash mismatch → terminate; inflight without compensate → ResumeCompensationGap; pre-1.0 migration path

### M2-T3 — GET /sessions/:id/state endpoint (§12.5.1, NEW spec)

- [ ] Endpoint: `GET /sessions/<session_id>/state`
- [ ] Response schema: `schemas/session-state-response.schema.json` (spec `6566707`)
- [ ] Bearer scope: `sessions:read:<session_id>` (granted by default on §12.6 session bootstrap alongside resolve + audit:read)
- [ ] Rate limit: 120 rpm per bearer
- [ ] Not-a-side-effect (MUST): no state advance, no StreamEvent, no audit write, no file write. Read-only in-memory observation; cache-refresh from disk at most once per request and NOT a mutation
- [ ] Response MUST reflect post-flush state (no stale pre-flush in-memory data; a crash-and-restart MUST observe the same state the endpoint reported)
- [ ] 400/401/403/404/429/503 per standard patterns
- [ ] Tests (6): happy path (pending + committed + compensated side_effects); not-a-side-effect (two reads byte-identical, workflow state unchanged); 404 unknown session; 403 wrong bearer; 429 rate limit; 503 pre-boot

### M2-T4 — Audit-sink three-state degradation (§10.5.1, SV-PERM-19)

- [ ] Implement the three-state machine per §10.5.1:
  - `healthy`: ship-acknowledged ≤ 1s for most recent record; no events emitted
  - `degraded-buffering`: ship failed ≤ 60s OR buffer < 1000 records; continue operation; fsync-backed local queue at `/audit/pending/`; `AuditSinkDegraded` StreamEvent emitted once per transition; `/ready` stays 200
  - `unreachable-halt`: ship failed > 60s OR buffer > 1000 records; refuse new Mutating/Destructive tool invocations with `PermissionDenied reason=audit-sink-unreachable`; permit ReadOnly; `AuditSinkUnreachable` StreamEvent emitted; `/ready` flips 503 reason=audit-sink-unreachable; on recovery flush buffer in order + verify chain resumes correctly + emit `AuditSinkRecovered` → return to healthy
- [ ] Accept the `SOA_RUNNER_AUDIT_SINK_FAILURE_MODE=<healthy|degraded-buffering|unreachable-halt>` env var (§12.5.2) to drive state deterministically for conformance testing. Production guard: refuse startup with env set + non-loopback bind (same pattern as clock injection L-01)
- [ ] Local buffer MUST be on same FS as `/audit/permissions.log` with atomic writes per §12.3
- [ ] Buffered records retain original timestamp; sink-issued timestamp applied on ship, not at buffer time
- [ ] Tests (10): state transitions healthy→degraded→unreachable; state transitions unreachable→healthy on recovery; ReadOnly permitted in unreachable-halt; Mutating rejected with audit-sink-unreachable; buffer persistence across restart; chain integrity after buffered-record flush; env-var test hook drives each state; production-guard refuses startup on non-loopback with env set; /ready flips 503/200 correctly; StreamEvent emission once-per-transition

### M2-T5 — External WORM sink shipping (§10.5 rule 4)

### M2-T6 — Tool Registry idempotency classification gate (§12.2, SV-SESS-05)

- [ ] Tool Registry load-time validation:
